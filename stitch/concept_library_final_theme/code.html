Understanding Async/Await

Estimated reading time: 12 minutes · Mastered by 1.2k developers

The Introduction

In JavaScript, operations like fetching data from an API or reading a file can take time. Instead of blocking the rest of your program, JavaScript handles these "asynchronously." Async/Await is a powerful syntax that allows you to write this asynchronous code so it reads like standard, sequential instructions.

psychology The Mental Model

Think of a high-end restaurant kitchen. An async function is like a chef starting a pot of water to boil.

timer

"I'm going to start this task (await), but I'm not going to stand here doing nothing. I'll prep the garnishes while the water heats up."

play_circle

Code Comparison

The Old Way (Promises)

fetchData()
  .then(response => {
    return response.json();
  })
  .then(data => {
    console.log(data);
  })
  .catch(err => {
    console.error(err);
  });

The Async/Await Way

async function getData() {
  try {
    const res = await fetchData();
    const data = await res.json();
    console.log(data);
  } catch (err) {
    console.error(err);
  }
}

Common Pitfalls

warning

Forgetting the 'await' keyword

The function will return a Promise object instead of the resolved data.

warning

Using 'await' in a non-async function

This will throw a SyntaxError. Await can only be used inside async contexts.

warning

Sequential execution instead of Parallel

Awaiting independent requests one-by-one can slow down your app significantly.
